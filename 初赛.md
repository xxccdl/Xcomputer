### 1. Demo 简介

**是什么：** Xcomputer 是一款 Windows 桌面 AI 自动化助手（Electron 桌面应用）。用户用自然语言输入指令，AI 自动规划步骤、调用工具操控电脑，完成从文件管理到系统操作的各类任务。

**面向谁：**
- **职场白领**：日常办公中有大量重复性电脑操作（整理文件、跨应用复制、数据录入等）
- **开发者**：需要批量代码搜索、终端命令执行、项目脚手架搭建
- **非技术用户**：不熟悉命令行和系统操作，希望用自然语言完成复杂任务
- **效率追求者**：希望用 AI 释放双手，专注于更有价值的创造性工作

**主要功能：**

#### 功能一：自然语言桌面自动化
一句话描述任务，AI 自动拆解并执行。支持打开应用、点击操作、文本输入、截图识别、窗口管理等 19+ 种桌面操作。

**使用示例：**
- "打开记事本写一首关于夏天的诗"
- "截取当前屏幕并保存到桌面"
- "把 D 盘 download 文件夹里的 PDF 都移动到 文档/资料 目录"

#### 功能二：编程模式 + 后台终端
专为开发者设计的编程模式，支持文件读写、代码搜索、终端命令执行。内置本地 Terminal 工具，可创建后台终端执行 npm、git、python 等命令，实时读取输出。

**使用示例：**
- "帮我在 src/components 下新建一个 Button 组件，用 TypeScript + Tailwind"
- "执行 npm run build 看看有没有编译错误"
- "搜索项目中所有使用了 useEffect 的文件"

#### 功能三：记忆系统 + 技能市场
- **Xmemory 长期记忆**：AI 自动记住你的偏好、习惯、常用操作，越用越懂你
- **技能系统**：上传或自动生成技能（如"安装 Chrome 浏览器"、"部署前端项目"），遇到相关任务时自动触发
- **XSkillHub 技能市场**：内置技能分享平台，可下载社区技能，也可发布自己的技能
- **子代理并行**：复杂任务自动拆分给多个子代理并行执行，效率翻倍
- **定时任务**：支持一次性/间隔/每日/每周定时执行，解放双手

> 🛡️ **安全保障**：所有高危操作（文件删除、注册表修改、进程终止）都会弹出确认对话框，你始终拥有最终控制权。

### 2. Demo 创作思路

**灵感来源：**

灵感来源于大模型 Agent 技术的成熟。既然 AI 已经能理解自然语言、能调用工具、能根据结果循环决策，那为什么不让它直接操控我们的电脑？

每次想完成一个多步骤的电脑任务，都要自己手动点击、输入、切换窗口、检查结果…… 这些机械重复的操作本应交给 AI。DeepSeek 等国产大模型的推理能力已经足够支撑复杂的工具调用循环，Windows-MCP 框架也提供了完整的桌面操控能力，两者结合就能实现"一句话操控电脑"的体验。

另一个灵感来源是 Cursor 等 AI 编程助手。它们让写代码的效率提升了 10 倍，但仅限于代码领域。我在想：能不能把这种"AI 助手 + 工具调用"的模式扩展到整个桌面？不只是写代码，而是所有电脑操作都能用自然语言完成？

于是有了 Xcomputer。

**想解决的问题：**

1. **重复操作耗时**：日常办公中大量重复性电脑操作（整理文件、填写表格、跨应用复制粘贴）耗费时间精力
2. **技术门槛高**：非技术用户面对系统维护、批量处理等任务无从下手，命令行、脚本编程门槛太高
3. **现有工具不够智能**：AutoHotkey 等自动化工具需要编程能力，RPA 软件学习成本高且不够灵活，无法应对非标准化场景
4. **跨应用协作困难**：很多任务需要在多个应用间切换（浏览器 → Excel → 邮件），手动操作容易出错且效率低下

**为什么做这个方向：**

我做了几个方向的取舍：

1. **选择桌面端而非云端**：虽然 Web 应用更容易分发，但桌面自动化必须在本地运行才能操控电脑。Electron 跨平台方案成熟，打包后用户下载即用。

2. **选择自然语言而非图形化拖拽**：图形化流程编辑器（如 Zapier、n8n）学习成本仍然高，而且只能处理预设好的流程。自然语言是最灵活的交互方式，用户不需要学习任何新东西，想说什么就说什么。

3. **选择 ReAct Agent 而非固定流程**：固定流程只能处理标准化任务，遇到稍微变化的场景就失效了。ReAct 模式下 AI 能自主思考、观察结果、调整策略，处理能力强得多，能应对各种意外情况。

4. **选择 Windows 优先**：Windows 是桌面市场份额最大的系统，Windows-MCP 工具链也最成熟。先把 Windows 体验做深做透，再考虑扩展到其他平台。

### 4. TRAE 实践过程

#### 整体开发流程

整个 Xcomputer 项目从 0 到 1 完全由 TRAE 辅助开发完成，整体流程如下：

```
需求沟通 → 架构设计 → 主进程搭建 → MCP 集成
     → AI 服务接入 → ReAct Agent 编排 → 前端界面开发
     → 记忆系统 → 技能系统 → 定时任务 → 悬浮球
     → 安装程序打包 → 宣传片制作 → 优化迭代
```

#### 关键开发步骤

**步骤一：项目初始化与架构搭建**
- 使用 TRAE 创建 Electron + React + TypeScript 项目
- 设计三栏 IDE 风格的深色主题界面
- 搭建主进程 / preload / 渲染进程三层架构

**步骤二：MCP 集成与 AI 服务**
- 集成 Windows-MCP 框架，实现 19 个桌面操作工具
- 接入 DeepSeek 双模型（V4-Flash 意图分类 + V4-Pro 任务执行）
- 实现 ReAct Agent 循环，支持最多 1000 轮自主决策

**步骤三：核心功能模块开发**
- 本地 File 工具：更强大的文件操作（递归列表、通配符搜索）
- 本地 Terminal 工具：后台终端管理，支持命令执行和输出读取
- Xmemory 记忆系统：自动提取用户偏好，上下文注入
- 技能系统 + XSkillHub：技能上传、下载、分享平台
- 子代理系统：支持 foreground/background 两种模式
- 定时任务：一次性/间隔/每日/每周自动执行

**步骤四：用户体验优化**
- 桌面悬浮球：实时显示 AI 状态，快捷唤起主窗口
- 工具预显示：AI 生成参数时立即显示加载动画和计时器
- 高危操作确认：删除/注册表/进程终止自动弹窗
- 安装程序现代化：浅色现代风格 NSIS 安装界面

**步骤五：宣传片与打包发布**
- Remotion 制作产品宣传片，BGM 节奏卡点
- electron-builder 打包，内置 Python + MCP 运行时
- 产品官网开发，提供下载入口
